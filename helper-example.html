Aurio Widget Integration Examples

This page demonstrates different ways to integrate the Aurio consultation widget using the helper script.

Method 1: Auto-initialization

The widget automatically initializes when a container with id="aurio-widget" is found.

<!-- Widget will auto-initialize in this container --> <div id="aurio-widget"></div> <!-- Include the helper script --> <script src="./aurio-widget.js"></script>

Method 2: Manual initialization

You can manually control when and where the widget initializes.

<div id="custom-widget-container"></div> <script src="./aurio-widget.js"></script> <script> // Manual initialization with custom options const widget = new AurioWidget({ containerId: 'custom-widget-container', autoInit: false }); // Initialize when ready widget.init(); </script>

Method 3: Custom configuration

You can customize the widget paths and behavior.

<script> const customWidget = new AurioWidget({ containerId: 'configured-widget', cssPath: './dist/widget.css', // Non-hashed CSS filename jsPath: './dist/widget.js', // Non-hashed JS filename autoInit: true }); </script>

Benefits of Using aurio-widget.js Helper

  • Plug-and-play: Just include the script and add a container
  • Error handling: Built-in error handling for failed loads
  • Customizable: Configure paths, container IDs, and initialization behavior
  • API methods: Programmatic control with init() and destroy() methods
  • Auto-styling: Automatically loads required CSS
  • No dependencies: Works without any external libraries
